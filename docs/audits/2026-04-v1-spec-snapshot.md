## 프로젝트 개요

AX(AI Experience) 기반 여행 플래너. 사용자가 여행 방향과 조건을 제시하면 AI가 숙소/항공편/일정을 검색·분석·생성하여 완성된 일정을 딜리버리한다.

## 목표

- 여행 플래닝 전 과정을 AI 서포트로 자동화
- 완성된 일정을 비개발자 동행자에게 즉시 공유 가능한 형태로 제공
- 1인 개발, 최소 비용으로 운영
- **모바일 최적화 필수** — 모든 산출물은 모바일에서 잘 보여야 함

## 타겟 사용자

| 역할 | 인원 | 도구 |
|------|------|------|
| 개발자 (나) | 1명 | Claude Desktop/Code, CLI |
| 동행자 (비개발자) | 2~5명 | GitHub 링크 열람 (모바일 브라우저) |

## 딜리버리

- **일정 원본**: 마크다운 (레포 `trips/` 디렉토리)
- **공유 형태**: GitHub 링크 (마크다운 렌더링 — 모바일 반응형)
- 수정 반영: push 즉시
- 웹앱은 v2 범위
- PDF는 v1 범위 제외 (필요 시 v2+에서 GitHub Actions로 추가)

## 레포 공개 설정

- 현재: private
- **public 전환 예정** — 동행자 링크 열람 + 포트폴리오 활용
- 전환 전 필수 작업:
  - 커밋 히스토리 내 토큰/시크릿 제거
  - API 키 로테이션 (기존 키 폐기 + 새 키 발급)
  - `.gitignore`에 `.env` 등 시크릿 파일 제외 확인

## 유저 플로우

### Phase 1 — 플래닝 (개발자 + 동행자)
1. 여행 조건 입력 (도시, 날짜, 예산, 스타일)
2. Claude가 Booking.com API로 호텔/항공 검색 + 분석
3. 중간 결과를 동행자에게 공유 (카톡 등)
4. 동행자 피드백 반영 → 재검색/수정
5. 숙소/항공/일정 확정
6. 마크다운 일정표 생성 → GitHub push

> **v1 제약**: 플래닝 단계에서 개발자가 Claude 실행 + GitHub push를 담당하므로 병목이 됨.
> 동행자 피드백은 카톡 등으로 수집하여 개발자가 반영하는 구조.
> v2 웹앱 도입 시 해소 예정.

### Phase 2 — 공유
7. GitHub 링크 전달 (카톡/메일)
8. 동행자 모바일 브라우저에서 일정 열람

### Phase 3 — 여행 중
9. GitHub 링크로 일정 확인 (모바일)

## 기술 스택

### API
- **Booking.com RapidAPI** (booking-com15 by DataCrawler)
  - 플랜: Pro ($8.99/월)
  - 한도: 기본 35,000회/월, getRoomListWithAvailability 2,000회/월
  - 엔드포인트: 48개 (호텔 20, 항공 7, 관광 6, 렌터카 8, 택시 2, 메타 5)
  - 리뷰·사진·시설·정책 모두 포함

### Claude 연동
- Claude Desktop/Code를 통한 로컬 사용
- Claude API 별도 비용 없음 (구독 범위 내)
- MCP 서버로 Booking.com API 연결

### 일정 관리
- 마크다운 파일 기반 (레포 `trips/{year}-{trip-name}/`)
- 기존 디렉토리 구조 유지

## 데일리 파일 포맷

기존 CLAUDE.md 규칙 준수:
1. 오늘의 요약 — 도시, 숙소, 이동, 예상 경비
2. 숙소 — 체크인/아웃, 가격, 상태, 리뷰 점수, 시설
3. 이동 — 도시 간 이동 수단, 시간, 비용, 예약 여부
4. 일정 — 시간대별 일정 + 예약 필요 여부
5. 투어/관광 상세 — 예약처, 비용, 소요시간
6. 식사 추천 — 장소, 메뉴, 가격대, 예약 필요 여부
7. 메모 — 팁, 주의사항
