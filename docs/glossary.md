# 용어사전 (Glossary)

본 프로젝트의 도메인 용어를 한국어·영어로 매핑한다. **사용자 화면에 노출되는 한국어가 정본**이고, 영어는 코드/스키마/API 식별자에서만 사용한다. 스펙 문서·PR 설명·릴리즈 노트·UI 카피는 모두 한국어 정본 용어로 작성한다.

## 역할 (Role)

| 한국어 (정본) | 영어 (코드) | 정의 |
|---|---|---|
| **주인** | `OWNER` | 여행을 생성하고 소유한 사용자. 여행당 1명. **호스트 권한을 포함**하며 추가로 소유권 행위(삭제·양도·공유 캘린더 연결/해제)를 가진다. |
| **호스트** | `HOST` | 여행을 함께 계획하는 편집 권한 동행자. 일정·활동 편집 + 공유 캘린더 sync 가능. |
| **게스트** | `GUEST` | 여행을 조회만 할 수 있는 동행자. |

> UI 예시: 동행자 목록의 "주인" 뱃지, "주인만 삭제할 수 있습니다" 에러 메시지, "동행자 초대" 버튼(spec 041 — 단일 진입점, 다이얼로그 안에서 호스트/게스트 역할별 초대 링크를 복사).

### 역할 포함 관계

**주인 ⊃ 호스트 ⊃ 게스트** (권한 포함). 데이터 모델(`TripRole` enum)은 상호 배타(한 사용자는 한 역할만 보유)이지만, 의미 계층상 주인은 호스트가 가진 모든 편집 권한을 가진다. v2.10.0(spec 023)부터 동행자 목록 UI에서 주인은 "주인" + "호스트" 두 뱃지를 병렬 표시해 이 포함 관계를 시각화한다.

### 권한 매트릭스

| 행위 | 주인 | 호스트 | 게스트 |
|---|:-:|:-:|:-:|
| 여행 조회 | ✓ | ✓ | ✓ |
| 일정·활동 편집 | ✓ | ✓ | ✗ |
| 호스트 초대 | ✓ | ✗ | ✗ |
| 게스트 초대 | ✓ | ✓ | ✗ |
| 공유 캘린더 sync | ✓ | ✓ | ✗ |
| 공유 캘린더 연결/해제 | ✓ | ✗ | ✗ |
| 여행 삭제·양도 | ✓ | ✗ | ✗ |

## 엔티티 (Entity)

| 한국어 (정본) | 영어 (코드) | 정의 |
|---|---|---|
| **여행** | `Trip` | 계획의 최상위 단위. 제목·기간·주인을 가진다. |
| **동행자** | `TripMember` | 여행에 참여한 사용자 + 역할. 호스트·게스트를 아우르는 상위 용어 — 초대 UI는 "동행자 초대" 단일 진입점으로 통합한다(spec 041). |
| **일정** | `Day` | 여행 기간 내 하루치 계획. (UI 상단 "일정" 섹션) |
| **활동** | `Activity` | 일정 내의 구체 항목(장소·시간). (UI "+ 일정 추가"의 대상) |
| **공유 캘린더** | `TripCalendarLink` | 여행당 1개. 주인 계정에 생성되어 전체 동행자가 외부 캘린더에서 공유 (v2.9.0+ 정본). |
| **내 구독** | `MemberCalendarSubscription` | 동행자 본인이 공유 캘린더를 자기 외부 캘린더 UI에 추가했는지 상태. |
| **이벤트 매핑** | `TripCalendarEventMapping` | 활동 ↔ 외부 캘린더 이벤트 1:1 대응 (v2.10.0+ 정본). 공유 캘린더에 귀속. |
| **레거시 캘린더 링크** | `GCalLink` | v2.8.0의 per-user 캘린더 모델. v2.9.0 이후 신규 쓰기 없음, 후속 contract에서 DROP 예정. |

## 상태 (State)

| 한국어 (정본) | 영어 | 정의 |
|---|---|---|
| **미연결** | not linked | 여행에 공유 캘린더가 아직 생성되지 않은 상태. 주인이 연결하기 전. |
| **연결됨** | linked | 여행에 공유 캘린더가 존재. |
| **권한 회수됨** | revoked | 외부 권한이 회수되어 동기화가 중단된 상태. |
| **추가됨** | added | 동행자가 공유 캘린더를 자기 외부 캘린더 UI에 추가한 상태. |
| **미추가** | not added | 공유 캘린더는 연결됐지만 동행자가 자기 UI에 아직 추가하지 않은 상태. |
| **건너뜀** | skipped | 동기화 시 사용자가 직접 수정한 이벤트로 판정되어 덮어쓰지 않은 상태. |

## 시간 (Time)

여행 일정 시간의 정본 모델. 상세 원칙은 헌법 VII(Calendar Time Model) 참조.

| 한국어 (정본) | 영어 (코드/표준) | 정의 |
|---|---|---|
| **부동 시간** | floating time (RFC 5545) | 타임존을 부착하지 않는 시각 표기. 기재된 벽시계 값(예: 5/8 09:00)이 불변이며, 보는 사람 위치와 무관하게 같은 숫자로 표시한다. 여행 일정 시간의 정본 모델. |
| **타임존 라벨** | `startTimezone` / `endTimezone` | 부동 시간이 어느 지역 기준 벽시계인지 가리키는 라벨. 표시 숫자를 바꾸지 않으며, 외부 캘린더 연동 시 단일 순간(UTC) 환산에만 쓰인다. |
| **절대시간** | UTC instant | 타임존 무관 단일 순간. 여행 일정에는 쓰지 않는다(관찰자 위치에 따라 표시가 달라져 현지 기준 표기와 어긋남). |

## 동작 (Action)

| 한국어 (정본) | 영어 (엔드포인트/함수) | 정의 |
|---|---|---|
| **공유 캘린더 연결** | create `TripCalendarLink` | 주인이 여행에 공유 캘린더를 생성·채택 + 동행자 권한 부여. |
| **공유 캘린더 해제** | unlink | 주인이 공유 캘린더 연결을 해제. 캘린더 자체는 외부 계정에 남는다. |
| **다시 반영하기** | sync | 여행의 최신 일정·활동을 공유 캘린더에 반영. |
| **내 구글 캘린더에 추가** | subscribe | 동행자가 공유 캘린더를 자기 구글 캘린더 UI에 추가. |
| **내 캘린더에서 제거** | unsubscribe | 동행자가 본인 UI에서만 제거. 권한은 유지. |
| **초대** | invite | 주인/호스트가 동행자를 여행에 참여시킨다. |
| **양도** | transfer ownership | 주인 역할을 다른 호스트에게 넘긴다. |

## 외부 · 일반

| 한국어 (정본) | 영어 | 정의 |
|---|---|---|
| **외부 캘린더** | external calendar | Google Calendar, Apple iCloud 등 제3자 캘린더 서비스. |
| **구글 캘린더** | Google Calendar | Google이 제공하는 캘린더. 공유 캘린더의 기본 채널. |
| **동의** | consent | OAuth scope 부여. |
| **권한** | ACL / access role | 외부 캘린더 상의 역할(reader/writer). |

## UI 시스템 — Viewport 분류 (반응형)

분류 기준은 viewport(브라우저 창) 폭입니다. 기기 종류가 아닌 폭으로 분기합니다 (태블릿 가로·폴더블·desktop 창 축소가 같은 viewport 폭이면 같은 카테고리로 처리합니다).

| 명칭 | viewport 폭 | 레이아웃 적용 |
|---|---|---|
| `mobile` | <1024px | stacked (세로 적층) |
| `desktop` | ≥1024px | split (좌·우 분할) 가능 |

* 명명은 일반 카테고리(mobile/desktop) 차용이지만 정의는 viewport 폭 범위입니다.
* `<1024px`에는 일반 스마트폰·태블릿 세로·desktop 창 축소가 모두 포함됩니다. UI는 이들을 구분하지 않습니다.
* 1인 개발·디자이너 1인 환경에서 다단계 breakpoint 분기는 유지보수 비용이 커서 2단으로 단순화합니다.

### 디자인 토큰과의 관계

`specs/026-responsive-layout`은 디자인 토큰 차원에서 4단(`breakpoint-mobile` / `-tablet` / `-desktop` / `-wide`)을 정의합니다. grid·container max-width·gap 같은 토큰은 4단을 그대로 참조합니다. 다만 **레이아웃 분기점(stacked↔split)은 1024px 단일**입니다.

* spec 026의 `tablet` 토큰은 디자인 토큰 차원에서만 존재합니다 (gap·padding 등). 레이아웃 분기점으로는 쓰지 않습니다.
* spec 026의 `wide` 토큰은 desktop 안의 컨테이너 max-width 적용 시점입니다.

### Tailwind 매핑

| 본 정본 명칭 | Tailwind prefix |
|---|---|
| `mobile` | 기본 클래스 (no prefix) |
| `desktop` | `lg:` (≥1024px) |

`md:`(≥768px)는 spec 026의 디자인 토큰 차원 분기에 쓰일 수 있지만 레이아웃 분기에는 쓰지 않습니다.

## 사용 원칙

1. **사용자 화면의 한국어가 정본이다.** UI 카피가 바뀌면 본 사전도 같은 PR에서 동시 업데이트한다.
2. **스펙·PR·이슈·릴리즈 노트는 한국어 정본 용어로 작성한다.** 영어 코드 식별자가 필요하면 `` `OWNER` ``처럼 백틱으로 감싸 인용한다.
3. **한 문서 안에서 혼용하지 않는다.** "주인"과 "오너"를 섞어 쓰지 말 것. 기존 문서에서 비정본 표기를 발견하면 해당 문서를 손댈 때 정본으로 교체.
4. **새 용어 도입은 본 사전 등재가 선행한다.** 코드·UI·스펙이 새 용어를 쓰기 전에 본 사전에 매핑을 추가한다.
5. **이미 릴리즈된 과거 스펙은 소급 교체하지 않는다.** 발견 시 주석/링크로 본 사전을 참조하도록 한다 — 기록물 보존 원칙.
