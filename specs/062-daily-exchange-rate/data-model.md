# Data Model: 원화 자동 근사 환산

## 신규 — `ExchangeRate` (근사 환율 캐시)

자동 확보한 일자·통화별 원화 근사 시세의 캐시. 여행자가 편집하지 않는다. 표시 보조 데이터이며 활동·금액 정본과 무관하다.

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | Int PK | |
| `date` | Date | 시세 기준 달력일 |
| `base` | VarChar(3) | 현지 통화 코드(ISO 4217). 예: `EUR` |
| `rateToKrw` | Decimal(18,6) | 현지 통화 1단위 = 원화 N |
| `fetchedAt` | Timestamptz | 확보 시각 |

- **유니크**: `(date, base)` — 일자·통화당 한 줄. 조회는 캐시 우선, 미스만 외부 확보 후 upsert.
- 기준 통화 `KRW`는 캐시하지 않는다(스스로 1).
- 마이그레이션: **schema-only**(신규 테이블, 백필 없음).

## 환산 규칙 (저장 아님 — 표시 시점 계산)

- 활동 금액(현화) × `rateToKrw(활동 일자, 활동 통화)` = 원화 근사값.
- 활동 통화가 `KRW`면 환율 없이 그대로 가산.
- 해당 (일자, 통화) 시세를 확보 못 하면 그 금액은 원화 합계에서 제외(`partial = true`).
- 일자 시세가 없으면 가장 가까운 가용 일자 시세로 대체 가능(참고치 허용 — 구현 선택).

## 기존 — 변경 없음

- `Activity`(현화 원본: `cost`/`currency`/`paymentTiming`)와 `Day`(`date`)는 그대로. 환산은 이 둘을 읽기만 한다.
