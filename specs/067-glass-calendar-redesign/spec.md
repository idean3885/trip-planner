# Feature Specification: 글래스 캘린더 재설계 + 글래스 카드 테두리 버그 수정

**Feature Branch**: `067-glass-calendar-redesign`
**Created**: 2026-07-18
**Status**: Draft
**Input**: User feedback: "글래스에 맞춰 캘린더 디자인 변경. 분리 배경색 제거, 사이즈 축소, 여행 기간 검은 선 대신 글래스 밴드. 카드 좌우 선 깨짐도 여전함."

## Clarifications

1. 카드 테두리 버그 — 글래스 카드가 `backdrop-filter` + `overflow:hidden` + `border-radius` 조합이라 Chrome이 필터 적용 전에 overflow를 잘라 세로 엣지·모서리에 아티팩트가 생긴다(WebKit #158807). 글래스 카드에서 overflow 클립을 걷어 해소한다.
2. 기간 표시 — 검은 밑줄(hard line) 대신 반투명 밴드(pill): 기간 전체에 반투명 틴트, 시작·끝을 둥글게. (사용자 선택)
3. 선택·오늘 — 솔리드 박스 제거. 선택 = 반투명 글래스 틴트 + 얇은 브랜드 링, 오늘 = 얇은 브랜드 블루 링. (사용자 선택)
4. 사이즈 — 셀이 정사각(`aspect-square`)이라 세로가 과대하고 월 그리드 아래 빈 영역이 크다. 셀 비율을 낮춰 축소한다.
5. 유지 — 요일 색 코딩(토=블루/일·주말=그린 글자), 팔레트·브랜드 이미지·기존 글래스 토큰.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 글래스 카드 테두리가 깨지지 않는다 (Priority: P1)

글래스 카드(활동 카드·컨테이너)의 좌우 세로 테두리와 모서리가 깨짐 없이 렌더된다.

**Independent Test**: 글래스 카드에서 overflow 클립이 제거되어(또는 마스크로 대체) 필터-클립 충돌이 없는지 소스 레벨로 검증.

**Acceptance Scenarios**:
1. **Given** 글래스 카드, **When** 렌더되면, **Then** 네 변·모서리에 렌더링 아티팩트가 없다.

---

### User Story 2 - 캘린더가 글래스 톤으로 보인다 (Priority: P1)

캘린더 선택일·오늘·여행 기간이 솔리드 필/검은 선 없이 글래스 톤(반투명 틴트·얇은 링·반투명 밴드)으로 보인다.

**Independent Test**: 선택/오늘/기간 스타일이 솔리드 필·`bg-primary` 밑줄 대신 글래스 토큰·링·밴드 클래스로 바뀌었는지 검증.

**Acceptance Scenarios**:
1. **Given** 캘린더, **When** 날짜를 선택하면, **Then** 반투명 글래스 틴트 + 얇은 링으로 표시된다(솔리드 박스 아님).
2. **Given** 캘린더, **When** 오늘을 보면, **Then** 얇은 브랜드 블루 링으로 표시된다(검은 박스 아님).
3. **Given** 여행 기간, **When** 렌더되면, **Then** 반투명 밴드로 이어지고 시작·끝이 둥글다(검은 밑줄 아님).

---

### User Story 3 - 캘린더가 콤팩트하다 (Priority: P2)

월 그리드가 화면을 과하게 차지하지 않고 아래 빈 영역이 줄어든다.

**Independent Test**: 셀 종횡비가 정사각보다 낮게(가로>세로) 설정됐는지 검증.

**Acceptance Scenarios**:
1. **Given** 월 캘린더, **When** 렌더되면, **Then** 셀 높이가 줄어 그리드가 콤팩트하고 하단 잉여 공간이 작다.

---

### Edge Cases
- 반투명 밴드 위 날짜 텍스트 대비 4.5:1 유지.
- 주(週) 경계에서 밴드가 자연스럽게 끊기고 각 주의 첫/끝 칸이 둥글게.
- 선택 + 오늘 + 기간이 겹치는 날의 표시 우선순위(선택 > 오늘 > 기간).

## Requirements *(mandatory)*

- **FR-001**: 글래스 카드는 `backdrop-filter`-클립 충돌 없이 네 변·모서리가 깨끗해야 한다.
- **FR-002**: 캘린더 선택일은 반투명 글래스 틴트 + 얇은 링으로 표시(솔리드 필 금지).
- **FR-003**: 오늘은 얇은 브랜드 블루 링으로 표시(검은 박스 금지).
- **FR-004**: 여행 기간은 반투명 밴드로 표시하고 시작·끝을 둥글게(검은 밑줄 금지).
- **FR-005**: 캘린더 셀 종횡비를 낮춰 그리드를 콤팩트하게, 월 그리드 하단 잉여 공간을 줄인다.
- **FR-006**: 밴드·틴트 위 날짜 텍스트는 WCAG AA(4.5:1)를 유지.
- **FR-007**: 색 값은 `:root` 토큰 경유(하드코딩 금지). 신규 캘린더 글래스 토큰은 `:root`에 정의.
- **FR-008**: 요일 색 코딩·팔레트·브랜드 이미지는 변경하지 않는다.

## Success Criteria *(mandatory)*

- **SC-001**: 글래스 카드 렌더링 아티팩트 0.
- **SC-002**: 선택·오늘·기간이 솔리드 필/검은 선 없이 글래스 표현으로 렌더.
- **SC-003**: 월 그리드 높이 축소(셀 종횡비 < 1).
- **SC-004**: 하드코딩 색 0건.
