# Quickstart: 여행 상세·목록 2분할 캘린더 레이아웃 + 연속 일자 가로 bar

**Feature**: `031-split-calendar-layout` | **Spec**: [spec.md](./spec.md) | **Plan**: [plan.md](./plan.md)

본 피처는 데스크탑(≥ 1024px) 한정 UI 변경. 자동 회귀 테스트 미도입 — 모든 Evidence 는 브라우저 수동 시각 검증.

## US1 — 여행 상세 2분할

### Scenario US1-1: viewport ≥ 1024px 에서 좌·우 두 컬럼만 보인다

`/trips/<id>` 진입 후 가운데 컬럼이 존재하지 않고 좌측 캘린더 / 우측 메타 두 컬럼만 보인다. 좌·우 폭이 시각적으로 비슷하다 (±10% 이내).

### Scenario US1-2: 일정 0 건 여행도 동일 레이아웃

`/trips/<id>` 일정 0 건 여행 진입 시에도 두 컬럼 비율이 동일하게 유지된다. 캘린더는 trip 기간 강조 없이 오늘만 강조된 상태.

### Evidence

- 자동 테스트: 해당없음 (UI 전용 — 자동 회귀 미도입)
- 수동 체크리스트:
  - [x] viewport 1440px 에서 좌·우 두 컬럼 폭 ±10% 이내 확인
  - [x] 가운데 영역(트립 체크박스 / 선택 날짜 일정 패널 / placeholder / "+일정 추가") 노출 0 건
  - [x] 일정 0 건 여행 동일 레이아웃
  - [x] "+ 일정 추가" 버튼이 H1 메타 행(호스트 초대·여행 나가기 옆)에서 발견 가능 (가운데 영역 제거 후 동선 보정)

## US2 — 연속 일자 가로 bar

### Scenario US2-1: ≥ 2 일 연속 일자가 가로 bar 로 연결

6/7 ~ 6/21 trip 등록 상태에서 6 월 캘린더 진입. 같은 주(예: 6/7 ~ 6/13) 안에서 가로 bar 가 끊김 없이 연결되어 보인다.

### Scenario US2-2: 다중 trip 색별 bar

두 trip 동시 표시 시 각 trip 이 서로 다른 색 bar 로 분리되어 보인다.

### Evidence

- 자동 테스트: 해당없음
- 수동 체크리스트:
  - [x] 단일 trip 6/7 ~ 6/13 같은 주에서 단색 가로 bar 연결 확인
  - [x] 주 경계(6/13 → 6/14) 에서 bar 가 행 단위로 분리되어 그려짐
  - [x] 2 개 trip 동시 표시 시 색별 bar 분리

## US3 — 셀 클릭 → Day 페이지

### Scenario US3-1: 등록된 셀 클릭 = 1 회 화면 전이

일정이 등록된 셀 클릭 시 `/trips/<id>/day/<dayId>` 로 즉시 이동. 중간 placeholder 노출 0 회.

### Scenario US3-2: 빈 셀은 클릭 비활성

일정 미등록 셀 hover 시 클릭 가능 시그널 없음. 클릭해도 라우팅 없음.

### Evidence

- 자동 테스트: 해당없음
- 수동 체크리스트:
  - [x] 등록 셀 클릭 시 즉시 `/trips/<id>/day/<dayId>` 라우팅
  - [x] 빈 셀 클릭 무반응

## US4 — 목록 통합 캘린더

### Scenario US4-1: 좌측 통합 캘린더에 모든 trip 일자 표시

2 개 trip 보유 사용자가 `/trips` 진입 시 좌측 캘린더에 두 trip 일자가 색별 bar 로 표시되고 우측에 두 카드 노출.

### Scenario US4-2: 셀 클릭 → 해당 trip 상세

등록된 셀 클릭 시 `/trips/<id>` 로 라우팅. 같은 셀에 ≥ 2 trip 겹치면 가까운 미래 trip 으로 이동.

### Evidence

- 자동 테스트: 해당없음
- 수동 체크리스트:
  - [x] /trips 진입 시 좌측 캘린더에 사용자의 모든 trip 일자 노출 (host·guest 무관)
  - [x] 통합 캘린더 셀 클릭 시 해당 trip 상세 라우팅
  - [x] 같은 셀에 2 trip 겹칠 때 가까운 미래 trip 으로 이동
