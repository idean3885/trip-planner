# Feature Specification: 여행 상세·목록 2분할 캘린더 레이아웃 + 연속 일자 가로 bar

**Feature Branch**: `031-split-calendar-layout`  
**Created**: 2026-05-28  
**Status**: Draft  
**Input**: 여행 상세(/trips/[id])와 여행 목록(/trips) 화면을 좌-캘린더(50%) / 우-메타(50%) 2분할로 재설계한다. 가운데 영역(트립 체크박스 + 선택 날짜 일정 패널 + placeholder + "+일정 추가") 제거. 캘린더에 연속된 여행 일자 가로 bar. 연관 이슈 #608.

## Clarifications

1. **빈 날 클릭 동작** — 일정이 등록되지 않은 캘린더 셀은 클릭 비활성 처리한다. WHY: 사용자가 비어 있는 셀을 눌렀을 때 라우팅 대상이 없으므로 의미 있는 화면 전이가 불가능하다. 일정 추가 동선은 별도(이슈 상세 페이지 내 "일정 추가" 버튼)로 유지한다.
2. **통합 캘린더 표시 범위** — /trips 좌측 통합 캘린더는 사용자가 host·guest 어떤 역할이든 속한 모든 trip 의 일자를 표시한다. WHY: 사용자의 "내가 관여한 모든 여행"을 한 눈에 보는 것이 우측 여행 목록과 짝을 이루는 가장 자연스러운 매핑이다.
3. **셀 멀티 trip 충돌 1차 처리** — 같은 셀에 여러 trip 이 겹치는 경우 클릭 시 가장 가까운 미래(또는 동일 날짜면 가장 최근 생성된) trip 상세로 이동한다. WHY: 1차 단순화. 충돌 해소 UX(셀 내 trip 선택 메뉴 등)는 후속 사이클로 미룬다.
4. **다중 trip 색 매핑** — 여러 trip 의 가로 bar / dot 색은 spec 029 Stage 2 에서 도입한 trip 별 색 dot 매핑 정책을 그대로 승계한다. WHY: 시각 일관성. 신규 색 규칙 도입은 비용 대비 가치 낮다.

## Metatag Conventions *(normative, inherited from PR #204)*

본 피처의 tasks.md·plan.md는 네 종 메타태그를 통해 후속 자동 검증과 연결된다:

- `[artifact: <path>|<path>::<symbol>]` — 산출 파일 식별자(drift 감사 기준)
- `[why: <short-tag>]` — 추적 그룹 키(plan↔tasks 커버리지·이슈 합산)
- `[multi-step: N]` — plan bullet이 다단 작업일 때 최소 매핑 태스크 수(N ≥ 2)
- `[migration-type: schema-only | data-migration]` — 마이그레이션 산출물 구분

형식 검증은 `.specify/scripts/bash/validate-metatag-format.sh`가 수행한다. 의미 검증은 각 US의 validator가 담당한다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 여행 상세에서 캘린더와 메타를 동시에 본다 (Priority: P1)

여행 상세에 진입한 사용자는 화면의 절반을 차지하는 캘린더로 여행 기간·일자별 일정 유무를 시각적으로 파악합니다. 화면의 나머지 절반에서 외부 캘린더 연결 상태와 동행자를 동시에 확인합니다. 가운데에 끼어 있던 안내 텍스트·트립 체크박스·"+일정 추가" 영역은 보이지 않습니다.

**Why this priority**: 현재 가독성 회귀의 직접적인 원인이 캘린더가 화면의 1/3 만 차지하고 가운데에 placeholder 가 남는 구조이다. 가운데 영역 제거 + 50% 캘린더만으로 사용자가 보고한 불만의 핵심을 해소한다.

**Independent Test**: /trips/[id] 페이지 단독으로 viewport ≥ 1024px 에서 좌·우 두 컬럼이 비슷한 폭으로 보이고 가운데 컬럼이 보이지 않는 것을 확인하면 검증된다.

**Acceptance Scenarios**:

1. **Given** 사용자가 일정 ≥ 1 건 등록된 여행에 진입했고 viewport ≥ 1024px **When** 화면이 렌더된다 **Then** 좌측 셀에 월간 캘린더가, 우측 셀에 외부 캘린더 동기화 카드 + 동행자 카드가 보이고 가운데 영역이 존재하지 않는다.
2. **Given** 같은 조건 **When** 사용자가 좌·우 컬럼의 폭을 시각적으로 비교한다 **Then** 좌측 캘린더 컬럼이 우측 메타 컬럼과 ±10% 이내 폭으로 보인다.
3. **Given** 일정 0 건인 여행 **When** 진입한다 **Then** 좌측 캘린더가 동일 폭으로 보이고 trip 기간 강조 없이 오늘 강조만 표시된다.

---

### User Story 2 — 캘린더에서 연속된 여행 일자를 한 눈에 본다 (Priority: P1)

사용자는 월간 캘린더 위에서 본인 여행의 연속된 일자를 가로 bar 로 인식한다. 멀티데이 여행(예: 7일 ~ 21일)이 끊김 없이 한 줄로 이어진 색 띠로 보여 일정 전체 윤곽을 즉시 파악한다.

**Why this priority**: 일자 단위 dot 만 있던 기존 표시는 사용자가 연속성을 직관적으로 파악하기 어렵다. 가로 bar 는 일정 그룹화 + 기간 인식을 동시에 충족한다.

**Independent Test**: 캘린더 단독으로 ≥ 2 일 이상 연속된 여행 일자가 가로 bar 로 연결되어 보이는지 확인하면 검증된다.

**Acceptance Scenarios**:

1. **Given** 한 여행이 6/7 ~ 6/21 등록 **When** 6월 캘린더가 렌더된다 **Then** 6/7 셀부터 6/21 셀까지 연속된 가로 bar 가 같은 행 안에서 끊김 없이 그려진다.
2. **Given** 같은 여행 **When** 캘린더 주가 바뀌는 경계(예: 6/13 토 → 6/14 일) **Then** bar 가 주 단위로 끊기되 양쪽 행 모두 연속된 일자임을 인식할 수 있도록 표시된다.
3. **Given** 두 개 이상 trip 동시 표시 **When** 캘린더가 렌더된다 **Then** 각 trip 이 서로 다른 색 bar 로 분리되어 어느 trip 인지 구분 가능하다.

---

### User Story 3 — 캘린더에서 일정으로 한 번에 진입한다 (Priority: P1)

사용자가 캘린더에서 일정이 있는 날짜 셀을 클릭하면 가운데 placeholder 안내를 거치지 않고 해당 Day 상세 페이지로 즉시 이동한다.

**Why this priority**: 가운데 영역을 제거하면서 기존 "셀 클릭 → 가운데 영역에 일정 표시" 동선이 사라진다. 즉시 Day 페이지 이동으로 대체해 사용자가 일정 상세까지 한 번의 클릭으로 도달한다.

**Independent Test**: 등록된 일자 셀을 클릭했을 때 /trips/[id]/day/[dayId] 로 라우팅되는지, 빈 셀은 클릭 비활성인지를 확인하면 검증된다.

**Acceptance Scenarios**:

1. **Given** 6/7 일자에 Day 가 등록 **When** 6/7 셀을 클릭 **Then** /trips/[id]/day/[dayId] 로 이동한다.
2. **Given** 6/9 일자에 Day 미등록 **When** 6/9 셀을 클릭 **Then** 라우팅이 일어나지 않고 셀 hover 시 클릭 가능 시그널이 없다.

---

### User Story 4 — 여행 목록에서 통합 캘린더로 전체 여행 흐름을 본다 (Priority: P2)

사용자가 /trips 에 진입하면 좌측에 본인이 속한 모든 여행의 일자가 한 캘린더에 모여 보인다. 우측에는 기존 여행 카드 목록이 그대로 노출된다. 캘린더 셀을 클릭하면 해당 여행 상세로 이동한다.

**Why this priority**: 여행 상세와 일관된 좌-캘린더 / 우-목록 레이아웃을 목록 페이지에도 적용한다. 사용자가 "다음 여행이 언제인지" 한 화면에서 즉시 파악한다. 상세 페이지 변경(P1) 이후 도입해도 단독 가치가 있다.

**Independent Test**: /trips 페이지 단독으로 좌측 캘린더에 사용자의 모든 trip 일자가 색별 bar 로 보이고 우측에 여행 카드 목록이 보이면 검증된다.

**Acceptance Scenarios**:

1. **Given** 사용자가 2 개의 여행에 속함 **When** /trips 에 진입 **Then** 좌측 캘린더에 두 여행의 일자가 서로 다른 색 bar 로 표시되고 우측에 두 여행의 카드가 나열된다.
2. **Given** 통합 캘린더에서 6/7 셀 클릭 (해당 일자에 trip A 등록) **When** 셀 클릭 **Then** /trips/A 로 이동한다.

---

### Edge Cases

- 같은 일자에 둘 이상의 trip 이 겹치는 셀 → Clarification 3 결정대로 1차는 가장 가까운 미래(또는 최근 생성) trip 으로 라우팅.
- 사용자가 속한 trip 0 건일 때 `/trips` 좌측 통합 캘린더 → 순수 월 달력만 표시(trip 기간 강조·dot·bar 없음). 우측 카드 목록은 빈 상태 안내("아직 등록된 여행이 없습니다") 노출.
- viewport < 1024px → 본 피처 범위 외. 기존 stacked 흐름(캘린더 위, 메타 아래) 유지.
- 일정 0 건 여행 → 캘린더가 빈 상태로 보이고 가로 bar 미표시. trip 기간 강조 없음.
- 가로 bar 가 한 주의 끝(토)에서 다음 주(일)로 이어질 때 → 같은 행 안에서만 그려지고 다음 행은 새로 시작되는 bar 로 그려진다(주 경계 단순 처리).
- 외부 캘린더 동기화 카드·동행자 카드가 우측에서 세로로 합쳐졌을 때 → 카드 높이 합산이 좌측 캘린더 높이보다 작으면 빈 공간 허용. 비대칭 회귀 없음.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 viewport ≥ 1024px 에서 /trips/[id] 본문을 좌측 컬럼(캘린더) / 우측 컬럼(메타) 두 열로만 표시해야 한다. 가운데 컬럼이 존재해서는 안 된다.
- **FR-002**: 시스템은 좌측 캘린더 컬럼 폭이 우측 메타 컬럼 폭과 ±10% 이내가 되도록 두 컬럼을 거의 균등하게 분배해야 한다.
- **FR-003**: 사용자는 캘린더 위에서 본인 여행의 연속된 일자를 같은 행 안에서 끊김 없이 가로 bar 로 인식할 수 있어야 한다.
- **FR-004**: 사용자가 둘 이상의 trip 일자를 동시에 표시하는 경우, 각 trip 의 가로 bar 가 서로 다른 색으로 구분되어야 한다.
- **FR-005**: 사용자가 일정이 등록된 캘린더 셀을 클릭하면 시스템은 해당 Day 페이지로 라우팅해야 한다.
- **FR-006**: 사용자가 일정이 등록되지 않은 빈 셀을 클릭해도 시스템은 라우팅을 수행하지 않아야 한다.
- **FR-007**: 시스템은 /trips 페이지 본문을 좌측 컬럼(통합 캘린더) / 우측 컬럼(여행 카드 목록) 두 열로 표시해야 하며, 좌측 캘린더는 사용자가 host·guest 어떤 역할이든 속한 모든 trip 의 일자를 색별 bar 로 표시해야 한다.
- **FR-008**: 사용자가 통합 캘린더의 일자 셀을 클릭하면 시스템은 해당 trip 상세로 라우팅해야 한다. 같은 셀에 둘 이상의 trip 이 겹치면 가장 가까운 미래(또는 동일 날짜면 가장 최근 생성) trip 으로 이동한다.
- **FR-009**: 시스템은 /trips/[id] 화면에서 트립 체크박스·선택 날짜 일정 패널·placeholder 안내·"+일정 추가" 버튼을 본문 가운데에 노출해서는 안 된다.
- **FR-010**: 시스템은 viewport < 1024px 에서 기존 stacked 흐름(캘린더 위, 메타 아래) 을 유지해야 한다. 본 피처는 데스크탑(≥ 1024px) 한정 변경이다.

### Key Entities

본 피처는 데이터 모델 변경 없이 기존 Trip / Day / TripCalendarLink / TripMember 엔티티를 화면에서 다르게 배치만 한다. 신규 엔티티 도입 없음.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: viewport ≥ 1024px 에서 /trips/[id] 좌측 캘린더 컬럼 폭이 본문 전체 폭의 45 % ~ 55 % 범위 안에 들어간다.
- **SC-002**: /trips/[id] 본문에서 가운데 컬럼(트립 체크박스·placeholder 안내·선택 날짜 일정 패널·"+일정 추가" 버튼) 요소가 0 개 노출된다.
- **SC-003**: ≥ 2 일 연속된 여행 일자가 같은 행에서 끊김 없이 가로 bar 로 연결되어 보인다 (시각 회귀 테스트 통과).
- **SC-004**: 일정이 등록된 셀 클릭이 1 회의 화면 전이로 해당 Day 상세에 도달한다 (중간 placeholder 거치는 단계 0 회).
- **SC-005**: /trips 좌측 통합 캘린더에 사용자가 속한 모든 trip 의 일자가 누락 없이 표시된다 (host·guest 무관).
