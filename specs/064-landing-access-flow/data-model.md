## 참조하는 런타임 상태

| 이름 | 출처 | 용도 |
|------|------|------|
| `isLoggedIn` | `auth()` 세션의 `session?.user?.id` 존재 여부 | 대문의 상태별 CTA·문구 분기, 리다이렉트 제거 후 진입 유도 결정 |
| `callbackUrl` | signin 페이지 `searchParams` | 로그인 후 목적지(지정 시 우선, 미지정 시 `/trips`) |

## 카피 데이터 (`src/lib/landing-content.ts`)

통합 대상 데이터 구조. 최종 카드 구성은 quickstart Evidence에서 확정하되, 다음을 보존한다:

- 대화·AI로 계획 (MCP·한 줄 설치 포함)
- 웹에서 직접 관리·협업 (여행→날짜→활동 3계층, 역할별 권한)
- 모바일 현장 활용 (현지 시간 표시, Apple 여행 캘린더 연동)
- 지출·가계부 (사전/현장 구분, 통화별 합산)

스키마가 아닌 정적 콘텐츠 상수이므로 마이그레이션 대상이 아니다.
