# Feature Specification: 외부 캘린더 가져오기 — 선택 + 시간·타임존 일괄 + 미저장 미리보기

**Feature Branch**: `033-calendar-sync-selection`  
**Created**: 2026-05-30  
**Status**: Draft  
**Input**: 외부 캘린더에서 일정을 가져와 목록으로 보여주고, 거기서 체크박스로 전체·부분 선택한다. 가져온 시점에는 아직 저장하지 않고 미리보기 상태로 두며, 시간 미정 일정 일괄 시작 시간·타임존 일괄·개별 수정을 거친 뒤 확정해야 정식 일정으로 저장된다. 확정 버튼은 상단에 고정하고 그 바로 밑에 시간 미정 일괄 시작·타임존 일괄을 둔다. 그 외 설정은 기본값(셀렉트 첫 값). 모바일 좁은 화면(아이폰 13 미니 등)에서 가로 스크롤이 생기지 않게 한다. 연관 이슈 #626.

## Clarifications

1. **가져오기 방향** — 외부 캘린더(Google·Apple)의 이벤트를 앱으로 가져오는 흐름입니다. 외부에서 읽어온 일정을 목록으로 보여주고, 사용자가 고른 것만 이 여행의 일정으로 들여옵니다. WHY: 동기화는 보통 그 여행에 한정되며, 외부 캘린더의 어떤 항목을 여행 일정으로 받을지 사용자가 골라야 합니다.
2. **가져온 시점에는 저장하지 않는다** — 가져온 일정은 정식 일정으로 저장되기 전의 미리보기 상태입니다. 선택·일괄 보정·개별 수정을 모두 마치고 확정해야 정식 일정으로 저장됩니다. WHY: 외부에서 읽어온 값을 바로 확정하면 잘못된 시간·타임존이 그대로 들어옵니다. 확정 전 보정 단계를 둡니다.
3. **시간·타임존 일괄과 개별 수정** — 미리보기 목록에서 시간 미정 일정에 일괄 시작 시간을 부여하고, 시간이 있는 일정은 시각 숫자를 유지한 채 타임존만 일괄 변경합니다. 개별 항목도 따로 수정할 수 있습니다. WHY: 외부 캘린더 이벤트는 타임존·시간 표기가 제각각이라 여행지 기준으로 한 번에 맞춘 뒤, 예외만 개별 손봅니다.
4. **타임존 일괄은 시간 숫자를 유지한다** — 타임존을 바꿔도 표시 시각 숫자(예: 09:00)는 그대로 두고 타임존 표기(IANA)만 교체합니다. 한국 09:00 → 현지 09:00 식입니다. WHY: 여행지 현지 시각 기준으로 맞추는 것이 사용자의 의도이며, 기존 floating-time 표시 관행(#232)과 정합합니다.
5. **체크박스 기본 선택 + 기타 기본값** — 가져온 목록은 모두 선택된 상태로 시작하고, 대상 캘린더 등 추가 설정은 셀렉트 첫 값을 기본 선택해 노출을 최소화합니다. WHY: 사용자가 결정할 것을 "어떤 항목을 들여올지"로 좁힙니다.
6. **모바일 가로 스크롤 금지** — 좁은 화면(아이폰 13 미니 등 360~375px 가용폭)에서 가져오기 화면에 가로 스크롤이 생기지 않아야 합니다. WHY: 가로 스크롤은 체크박스·시간·제목을 한 줄에 욱여넣어 발생하며, 좁은 화면에서 조작을 어렵게 합니다.

## 캘린더 시간 규칙 (헌법 VII 적용)

여행 일정 시간은 부동 시간(floating time)으로 다룬다 — 헌법 VII Calendar Time Model이 정본이며 본 피처의 시간·타임존 처리는 이를 따른다. 용어는 [glossary 시간 섹션](../../docs/glossary.md) 참조.

- 가져온 항목의 표시 시각은 벽시계 값을 그대로 노출한다(관찰자 타임존 환산 없음).
- **타임존 일괄**은 타임존 라벨만 교체하고 표시 시각 숫자는 유지한다(FR-009 근거). 타임존이 표시를 바꾸지 않는 라벨이기 때문이다.
- **시간 미정 일괄 시작**은 벽시계 시작 값을 부여하는 것이다(FR-008).
- 외부에서 가져온 절대시간(UTC)·TZID 이벤트는 이 여행의 부동 시간으로 받아들이며, 어느 지역 기준 벽시계로 고정할지는 타임존 라벨로 정한다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 외부 캘린더에서 가져온 일정을 골라 들여온다 (Priority: P1)

사용자는 외부 캘린더에서 가져온 일정을 목록으로 봅니다. 각 항목에 체크박스가 있어 전체 또는 일부를 선택하고, 확정하면 선택한 항목만 이 여행의 일정으로 들어옵니다. 진입 시 모두 선택돼 있어 빼고 싶은 것만 해제하면 됩니다.

**Why this priority**: 외부 캘린더의 어떤 항목을 여행 일정으로 받을지 고르는 것이 가져오기의 본질입니다.

**Independent Test**: 가져온 목록에서 일부 체크를 해제하고 확정하면, 선택된 항목만 여행 일정으로 저장되고 해제한 항목은 저장되지 않는지 확인하면 검증된다.

**Acceptance Scenarios**:

1. **Given** 외부 캘린더에 여러 이벤트가 있는 연결 상태 **When** 가져오기 화면에 진입 **Then** 외부 이벤트가 목록으로 나열되고 모두 선택된 상태로 보인다.
2. **Given** 가져온 목록 **When** 일부 항목 체크를 해제하고 확정 **Then** 선택된 항목만 여행 일정으로 저장된다.
3. **Given** 가져온 목록 **When** 전체 선택 토글을 끈다 **Then** 모든 항목 체크가 해제되고, 다시 켜면 모두 선택된다.

---

### User Story 2 — 확정 버튼과 일괄 설정이 상단에 고정된다 (Priority: P1)

사용자는 가져온 항목이 많아도 하단으로 스크롤하지 않고 상단에서 확정 버튼과 일괄 설정(시간 미정 일괄 시작, 타임존 일괄)을 바로 씁니다. 목록만 스크롤되고, 확정 버튼과 일괄 설정은 항상 보입니다.

**Why this priority**: 자주 누르는 확정 버튼과 일괄 설정이 하단으로 밀리면 조작 거리가 깁니다. 일정이 많을수록 더 멀어집니다.

**Independent Test**: 가져온 항목이 화면 높이를 넘는 경우 목록을 아래로 스크롤해도 확정 버튼과 일괄 설정 영역이 계속 보이는지 확인하면 검증된다.

**Acceptance Scenarios**:

1. **Given** 가져온 항목이 화면을 넘는 경우 **When** 목록을 아래로 스크롤 **Then** 확정 버튼과 일괄 설정 영역이 상단에 계속 보인다.
2. **Given** 가져오기 화면 **When** 화면을 본다 **Then** 확정 버튼 바로 아래에 "시간 미정 일괄 시작"과 "타임존 일괄"이 함께 위치한다.

---

### User Story 3 — 미저장 상태에서 시간·타임존을 일괄·개별 보정한 뒤 확정한다 (Priority: P1)

사용자는 가져온 항목을 저장하기 전에 보정합니다. 시간이 없는 항목에 일괄 시작 시간을 부여하고, 시간이 있는 항목은 시각 숫자를 유지한 채 타임존만 현지로 일괄 변경합니다. 예외 항목은 개별로 따로 수정합니다. 확정하면 보정된 값으로 정식 일정이 저장됩니다.

**Why this priority**: 외부 이벤트의 시간·타임존을 그대로 받으면 어긋납니다. 확정 전 보정이 가져오기 품질을 좌우합니다.

**Independent Test**: 가져온 미리보기에서 일괄 시작 시간·타임존을 적용하고 일부를 개별 수정한 뒤 확정하면, 보정된 값으로 일정이 저장되는지 확인하면 검증된다.

**Acceptance Scenarios**:

1. **Given** 시간 미정 항목이 포함된 미리보기 **When** 일괄 시작 시간을 적용 **Then** 시간 미정 항목에만 그 시작 시간이 들어가고, 이미 시간이 있는 항목은 바뀌지 않는다.
2. **Given** 시간이 있는 항목들 **When** 타임존 일괄을 현지로 적용 **Then** 표시 시각 숫자는 그대로이고 타임존 표기만 바뀐다.
3. **Given** 미리보기 항목 **When** 개별 항목을 수정 **Then** 그 항목만 바뀌고 다른 항목·미저장 상태는 유지된다.
4. **Given** 일괄·개별 보정을 마친 상태 **When** 확정 **Then** 보정된 값으로 선택 항목이 정식 일정으로 저장된다.

---

### User Story 4 — 좁은 모바일 화면에서 가로 스크롤 없이 본다 (Priority: P2)

아이폰 13 미니 같은 좁은 화면(가용폭 360~375px) 사용자는 가져오기 화면을 가로 스크롤 없이 봅니다. 체크박스·시간·제목이 한 항목 안에서 줄을 적절히 나눠 가로로 넘치지 않습니다.

**Why this priority**: 가로 스크롤은 좁은 화면에서 조작을 어렵게 합니다. 데스크탑·태블릿(P1) 동작 정리 후 좁은 폭 최적화를 적용해도 단독 가치가 있습니다.

**Independent Test**: 가용폭 375px 이하에서 가져오기 화면에 가로 스크롤바가 생기지 않고 모든 항목이 폭 안에 들어오는지 확인하면 검증된다.

**Acceptance Scenarios**:

1. **Given** 가용폭 375px 이하 화면 **When** 가져오기 화면을 연다 **Then** 가로 스크롤이 생기지 않는다.
2. **Given** 긴 제목의 항목 **When** 좁은 화면에서 본다 **Then** 제목이 잘리거나 줄바꿈되어 가로로 넘치지 않는다.

---

### Edge Cases

- 외부 캘린더에 가져올 이벤트가 0건 → 빈 상태 안내. 확정 버튼 비활성.
- 선택된 항목이 0건일 때 확정 → 저장하지 않음(비활성 또는 안내).
- 캘린더 미연결 상태 → 연결 안내를 먼저 보이고 가져오기는 연결 후 가능.
- 시간 미정 항목이 0건일 때 "시간 미정 일괄 시작" → 적용 대상 없음(무동작 또는 비활성).
- 이미 이 여행에 들어온 것과 겹치는 외부 이벤트 → 중복 표시 정책은 plan에서 결정(가정: 기존 가져오기 중복 처리 승계).
- 확정 도중 일부 항목 저장 실패 → 실패 항목 알림, 성공분 유지(부분 성공 허용).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 외부 캘린더에서 가져온 일정을 목록으로 표시하고 각 항목에 체크박스를 제공해야 한다.
- **FR-002**: 시스템은 가져온 목록을 진입 시 모두 선택된 상태로 표시해야 한다.
- **FR-003**: 사용자는 전체 선택 토글과 개별 체크박스로 들여올 항목을 전체·부분 선택할 수 있어야 한다.
- **FR-004**: 시스템은 가져온 항목을 확정 전까지 정식 일정으로 저장하지 않고 미리보기 상태로 유지해야 한다.
- **FR-005**: 시스템은 확정 시 선택된 항목만 정식 일정으로 저장하고, 선택되지 않은 항목은 저장하지 않아야 한다.
- **FR-006**: 시스템은 확정 버튼과 일괄 설정 영역(시간 미정 일괄 시작·타임존 일괄)을 화면 상단에 고정해, 목록을 스크롤해도 함께 보이도록 해야 한다.
- **FR-007**: 시스템은 확정 버튼 바로 아래에 "시간 미정 일괄 시작"과 "타임존 일괄"을 배치해야 한다.
- **FR-008**: 사용자가 일괄 시작 시간을 적용하면 시스템은 시간이 미정인 항목에만 그 시작 시간을 부여하고, 이미 시간이 있는 항목은 변경해서는 안 된다.
- **FR-009**: 사용자가 타임존 일괄을 적용하면 시스템은 항목의 표시 시각 숫자를 유지한 채 타임존 표기만 변경해야 한다.
- **FR-010**: 사용자는 미리보기 항목을 개별로 수정할 수 있어야 하며, 개별 수정은 그 항목에만 적용되어야 한다.
- **FR-011**: 시스템은 대상 캘린더 등 추가 설정 항목을 기본값(셀렉트 첫 값)으로 자동 선택하고 노출을 최소화해야 한다.
- **FR-012**: 시스템은 선택된 항목이 0건이면 확정(저장)을 수행하지 않아야 한다.
- **FR-013**: 시스템은 가용폭 375px 이하 좁은 화면에서 가져오기 화면에 가로 스크롤이 생기지 않도록 항목 레이아웃을 구성해야 한다.

### Key Entities

본 피처는 기존 `Trip` / `Day` / `Activity` / `TripCalendarLink` 엔티티와 외부 캘린더 가져오기 미리보기 개념을 사용한다. 가져온 미리보기 항목은 확정 시 `Activity`(필요 시 `Day` 동반 생성)로 저장된다. 미리보기 항목의 시간·타임존(`startTime` / `startTimezone`)은 확정 전 보정 대상이다. 미리보기 항목의 보관 위치(클라이언트 임시 / 기존 ActivityDraft 재사용)와 스키마 영향은 plan 단계에서 결정한다(현재 가정: 스키마 변경 없음).

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 가져오기 화면 진입 시 외부 이벤트가 목록으로 노출되고 전부 선택돼 있다.
- **SC-002**: 일부를 해제하고 확정하면 선택분만 정식 일정으로 저장된다(미선택 0건 저장).
- **SC-003**: 가져온 항목이 화면을 넘어도 확정 버튼과 일괄 설정 영역이 끝까지 보인다(스크롤로 사라지는 단계 0회).
- **SC-004**: 일괄 시작 시간 적용 후 시간 미정 항목에 시간이 채워지고, 시간 있던 항목은 변경되지 않는다.
- **SC-005**: 타임존 일괄 적용 후 항목의 표시 시각 숫자가 변하지 않고 타임존만 바뀐다.
- **SC-006**: 가용폭 375px 이하에서 가져오기 화면에 가로 스크롤이 발생하지 않는다.
