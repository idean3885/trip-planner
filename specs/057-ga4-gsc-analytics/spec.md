## Clarifications

초안 작성 중 드러난 모호 지점을 결정으로 봉합한다. WHAT/WHY 수준이며 구체 수단은 plan에서 정한다.

1. **분석 도구 = 시장 표준 단일 채택** — 사용 행태 분석 도구로 시장에서 가장 보편적인 표준 도구를 단일 채택하고, 대안 도구들은 보류한다. WHY: 차별화가 아니라 협업·채용에서 통하는 대세 적합성을 기준으로 고른다. 깊은 세션 분석이 필요해지면 후속에서 재검토한다. 구체 도구명과 선택 근거는 plan·ADR에서 다룬다.
2. **검색 노출은 최소만** — 공개 페이지(랜딩·소개·문서)만 sitemap·색인 대상으로 두고, 로그인 뒤 앱 본체는 색인에서 제외한다. WHY: 앱 가치의 대부분이 로그인 뒤 동적 데이터라 검색 색인 가치가 낮다. 검색 유입을 본격 노릴 의도가 생기면 후속에서 확장한다.
3. **측정 ID·검증값 미설정 시 분석/검증은 비활성, 앱은 정상 동작** — 외부 콘솔에서 발급하는 측정 ID·검증값은 환경 설정값으로 주입한다. 값이 없으면 분석·검증 태그가 붙지 않고 앱 기능은 그대로 동작한다. WHY: 콘솔 작업(계정 생성·발급·등록)은 사람이 하는 단계라, 코드는 값 유무와 무관하게 안전해야 한다.
4. **개인정보 고지는 최소, 동의 배너는 후속** — 분석이 쿠키를 쓴다는 사실을 개인정보 처리 고지에 반영하고, IP 식별 최소화 등 기본 보호를 적용한다. 별도 동의 배너 UI는 본 피처 범위 밖(후속)이다. WHY: 사용자가 내부자 중심인 초기 단계라 고지 우선으로 시작하고, 외부 유입이 늘면 동의 UX를 보강한다.
5. **앱 이름 정본 = "우리의 여행"** — 사용자 화면에서 프로젝트 코드명(영문 식별자)을 노출하지 않는다. 화면에 이름이 필요한 자리에는 앱 이름 "우리의 여행"을 쓰거나, 문맥상 자명하면 이름을 생략한다. WHY: 코드명은 개발 식별자일 뿐 제품 이름이 아니다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 가져오기 화면 카피 간소화 (Priority: P1)

사용자가 외부 캘린더 가져오기 화면을 볼 때, 프로젝트 코드명과 "외부 캘린더에 쓰지 않는다 / 정본은 ~이다" 같은 기술·내부 설명이 보이지 않는다. 화면은 무엇을 하는 화면인지 한눈에 알 수 있는 제목과 한 줄 설명만 보여준다.

**Why this priority**: 사용자가 직접 지적한 불편이고, 코드명 노출은 제품 완성도를 떨어뜨린다. 수정이 작고 즉시 가치가 있다.

**Independent Test**: 가져오기 화면과 관련 안내에서 코드명·"쓰지 않음/정본" 설명이 사라지고 제목 + 한 줄 설명만 보이는지 확인.

**Acceptance Scenarios**:

1. **Given** 가져오기 화면, **When** 화면을 연다, **Then** 프로젝트 코드명이 어디에도 보이지 않는다.
2. **Given** 가져오기 화면, **When** 화면을 본다, **Then** "외부 캘린더에 쓰지 않습니다 / 정본은 ~입니다" 류의 기술·내부 설명 박스가 보이지 않는다.
3. **Given** 가져오기 화면, **When** 화면을 본다, **Then** 제목과 한 줄 설명만으로 무엇을 하는 화면인지 알 수 있다.

---

### User Story 2 - 사용 행태 분석(GA4) 수집 (Priority: P1)

운영자(개발자 본인)가 공개된 서비스에서 사용자가 어떤 경로로 들어와 어떤 핵심 행동(가입, 여행 생성, 외부 캘린더 가져오기 실행)을 하는지 집계로 확인할 수 있다.

**Why this priority**: "어떤 사용자가 어떻게 쓰는지"를 보는 분석 기반이 이 마일스톤의 핵심 목적이다.

**Independent Test**: 측정 ID가 설정된 환경에서 페이지 방문과 핵심 전환 행동이 분석 대시보드에 집계되는지 확인. 측정 ID가 없는 환경에서는 분석 태그가 붙지 않고 앱이 정상 동작하는지 확인.

**Acceptance Scenarios**:

1. **Given** 측정 ID가 설정된 환경, **When** 사용자가 페이지를 방문한다, **Then** 페이지뷰가 집계된다.
2. **Given** 측정 ID가 설정된 환경, **When** 사용자가 여행 생성·외부 캘린더 가져오기 등 핵심 행동을 한다, **Then** 해당 전환 이벤트가 집계된다.
3. **Given** 로그인 사용자, **When** 행동을 한다, **Then** 그 행동이 익명 사용자 식별자로 연결되어 같은 사용자의 흐름으로 묶인다(개인정보는 전송하지 않음).
4. **Given** 측정 ID가 설정되지 않은 환경, **When** 앱을 사용한다, **Then** 분석 태그가 붙지 않고 모든 기능이 정상 동작한다.

---

### User Story 3 - 검색 노출 최소 등록 (Priority: P3)

운영자가 검색엔진에 공개 페이지(랜딩·소개·문서)만 노출되도록 등록하고, 로그인 뒤 앱 본체는 검색에 색인되지 않게 한다.

**Why this priority**: 동적 앱이라 검색 색인 가치가 낮다. 거의 비용이 없어 열어두지만 비중은 낮다.

**Independent Test**: 공개 페이지가 사이트맵에 포함되고, 로그인 뒤 페이지는 색인 제외로 표시되며, 검색엔진 소유 확인 값이 설정되면 적용되는지 확인.

**Acceptance Scenarios**:

1. **Given** 사이트맵, **When** 내용을 본다, **Then** 공개 페이지(랜딩·소개·문서)만 포함되고 로그인 뒤 경로는 빠진다.
2. **Given** 로그인 뒤 앱 경로, **When** 검색엔진 색인 정책을 본다, **Then** 색인 제외로 표시된다.
3. **Given** 검색엔진 소유 확인 값이 설정된 환경, **When** 페이지를 본다, **Then** 확인 값이 적용된다(미설정 시 미적용, 앱 정상).

---

### Edge Cases

- 측정 ID·검증값이 비어 있어도 페이지 로드·핵심 기능이 영향받지 않아야 한다.
- 분석 태그 로드가 실패해도 앱 동작이 중단되지 않아야 한다.
- 비로그인 방문자도 페이지뷰는 집계되되 사용자 식별자는 비로그인 상태로 다뤄진다.
- 코드명이 사용자 화면 외(개발 식별자·내부 상수)에 남는 것은 본 피처 대상이 아니다(사용자 노출 텍스트만 정리).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 가져오기 화면과 관련 사용자 안내에서 프로젝트 코드명 노출을 제거해야 한다(필요 시 앱 이름 "우리의 여행"으로 대체).
- **FR-002**: 가져오기 화면에서 "외부 캘린더에 쓰지 않음 / 정본은 ~" 류의 기술·내부 설명 박스를 제거하고 제목 + 한 줄 설명만 남겨야 한다.
- **FR-003**: 시스템은 공개 서비스의 페이지뷰를 집계해야 한다.
- **FR-004**: 시스템은 핵심 전환 행동(여행 생성, 외부 캘린더 가져오기 실행)을 이벤트로 집계해야 한다. (가입 시점의 정밀 집계는 서버 측 작업이 필요해 후속으로 둔다 — 로그인 사용자 활동은 페이지뷰 + 익명 식별자 연결로 파악한다.)
- **FR-005**: 시스템은 로그인 사용자의 행동을 개인정보가 아닌 익명 식별자로 연결해야 한다.
- **FR-006**: 측정 ID가 설정되지 않은 환경에서는 분석 태그가 붙지 않고 앱이 정상 동작해야 한다.
- **FR-007**: 분석이 쿠키를 사용한다는 사실을 개인정보 처리 고지에 반영해야 한다(동의 배너는 범위 밖).
- **FR-008**: 시스템은 공개 페이지(랜딩·소개·문서)만 사이트맵에 포함해야 한다.
- **FR-009**: 시스템은 로그인 뒤 앱 본체를 검색 색인에서 제외해야 한다.
- **FR-010**: 검색엔진 소유 확인 값이 설정되면 적용하고, 미설정 시 미적용하며 앱은 정상 동작해야 한다.
- **FR-011**: 분석·검색 노출 도구 선택 근거(GA4 채택, 대안 보류, 동적 앱에서 검색 노출의 위치)를 결정 기록으로 남겨야 한다.

### Key Entities *(include if feature involves data)*

- **방문/이벤트(집계)**: 외부 분석 서비스에 적재되는 익명 집계 데이터. 페이지뷰 + 핵심 전환 이벤트. 개인정보를 포함하지 않는다.
- **익명 사용자 식별자**: 로그인 사용자를 개인정보 없이 같은 흐름으로 묶기 위한 내부 식별자.
- **공개 페이지 목록**: 사이트맵·색인 대상이 되는 공개 경로 집합(랜딩·소개·문서).

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 가져오기 화면 사용자 노출 텍스트에 프로젝트 코드명이 0건이다.
- **SC-002**: 가져오기 화면의 기술·내부 설명 박스가 0개이고, 제목 + 한 줄 설명만 남는다.
- **SC-003**: 측정 ID가 설정된 환경에서 페이지뷰와 2종 핵심 전환 이벤트(여행 생성·가져오기 실행)가 집계된다.
- **SC-004**: 측정 ID 미설정 환경에서 분석 태그 0개, 핵심 기능 회귀 0건.
- **SC-005**: 사이트맵에 로그인 뒤 경로가 0건 포함되고, 공개 페이지는 모두 포함된다.
- **SC-006**: 도구 선택 근거가 결정 기록 문서로 남는다.
