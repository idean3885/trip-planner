# Data Model: 사용자 분석(GA4)·검색 노출(GSC) + 카피 간소화

**Feature**: 057-ga4-gsc-analytics | **Date**: 2026-06-05

**DB 스키마 변경 없음. 마이그레이션 없음.** 분석 데이터는 외부 GA4로 적재되며 앱 DB에 저장하지 않는다. 아래는 개념 엔티티다.

## 외부 적재(앱 DB 아님)

| 엔티티 | 설명 | 비고 |
|--------|------|------|
| 페이지뷰 | 공개 서비스 방문 집계 | `GoogleAnalytics` 자동 수집 |
| 전환 이벤트 | `trip_created`, `calendar_import`, `sign_in` | 클라이언트 `track()` 전송 |
| 익명 User-ID | 로그인 사용자를 같은 흐름으로 묶는 내부 식별자 | 개인정보(이메일·이름) 미포함 |

## 환경 설정값(코드 입력 아님)

| 키 | 용도 | 미설정 시 |
|----|------|-----------|
| `NEXT_PUBLIC_GA_ID` | GA4 측정 ID(`G-XXXX`) | 분석 태그 미렌더, 앱 정상 |
| 검색 소유 확인값 | 검색엔진 소유 확인 메타 | 메타 미적용, 앱 정상 |

## 공개/비공개 경로(색인 정책)

| 구분 | 경로 | 색인 |
|------|------|------|
| 공개 | `/`, `/about`, `/docs` | sitemap 포함 + 색인 허용 |
| 비공개(앱 본체) | `/trips/*`, `/settings/*`, `/day/*` 등 | robots disallow + meta noindex |

## 이벤트 파라미터 원칙

- `user_id`: 내부 사용자 식별자(난수성 ID). 이메일·이름 등 PII 전송 금지.
- 이벤트 파라미터에 여행 제목·활동 내용 등 콘텐츠 PII를 넣지 않는다(집계 목적 메타만).
