# Feature Specification: 모바일 캘린더 스크롤 접힘

## 배경

모바일에서 일정을 보려고 아래로 스크롤하면 캘린더가 화면 상단에 고정되지만 월간(여러
줄)이 그대로 남아 일정 영역을 좁힌다. 경계 멈춤(스크롤 정지)은 포기했으나, 스크롤 시
캘린더가 주간으로 접혀 공간을 넓게 쓰는 동작은 살린다. 좁은 폭 기기에서도 깨지지
않아야 한다.

## User Scenarios

### US1 — 스크롤 시 주간 자동 접힘 (P1)

모바일에서 아래로 스크롤해 캘린더가 화면 상단에 고정되는 시점에 캘린더가 월간에서
주간(선택 주 1줄)으로 접힌다. 맨 위로 돌아오면 월간으로 복원된다.

**Acceptance**
- 아래로 스크롤해 캘린더가 상단 고정되면 주간으로 접힌다.
- 맨 위로 복귀하면 월간으로 돌아온다.
- 접힌 주간에서도 선택 주는 현재 선택 일자를 따른다.
- 데스크탑(2분할)에는 영향이 없다.

### US2 — 좁은~큰 폭 반응형 (P1)

좁은 폭 기기(약 320px)부터 그보다 큰 기기까지 캘린더와 주간 스트립이 깨지지 않는다.

**Acceptance**
- 약 320px 폭에서 월간 셀·주간 스트립이 가로로 넘치거나 겹치지 않는다.
- 더 큰 폭에서도 같은 레이아웃이 유지된다.

## Requirements

- **FR-001** 모바일에서 캘린더가 스크롤로 상단 고정되면 주간으로 자동 접는다.
- **FR-002** 스크롤이 맨 위로 돌아오면 월간으로 복원한다.
- **FR-003** 접힘 상태는 현재 선택 일자의 주를 보여준다.
- **FR-004** 데스크탑 레이아웃에는 영향을 주지 않는다.
- **FR-005** 약 320px 폭에서 캘린더·주간 스트립 레이아웃이 유지된다.
- **FR-006** 더 큰 폭에서도 레이아웃이 유지된다.

## Out of Scope

- 경계 멈춤(스크롤 정지) 복원 — 별건으로 보류.
- 데스크탑 캘린더 동작 변경.

## 용어

- **접힘(주간)**: 월간(여러 줄) 대신 선택 주 1줄만 표시하는 압축 상태.
- **상단 고정(sticky stuck)**: 스크롤로 캘린더가 화면 맨 위에 붙어 고정된 상태.
