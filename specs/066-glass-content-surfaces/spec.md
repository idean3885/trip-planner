# Feature Specification: 글래스모피즘 확장 — 여행 상세 콘텐츠 + 카드 테두리 선명화

**Feature Branch**: `066-glass-content-surfaces`
**Created**: 2026-07-18
**Status**: Draft
**Input**: User feedback: "여행 상세 화면에 글래스가 안 보인다. 활동 카드·주간 달력까지 넣어달라. 카드 좌우 테두리가 흐리게 보인다."

## Clarifications

1. 확장 범위 — v4.8.0(spec 065)이 성능 우려로 불투명 유지했던 여행 상세 콘텐츠(활동 카드·주간 달력 래퍼)까지 글래스를 확장한다. 화면 대부분을 차지하는 표면이라, 여기까지 적용해야 "전면 적용"이 실제로 보인다.
2. 성능 — 다수 활동 카드가 스크롤 중 동시에 블러되는 위험은 실기기 검증으로 확인하고, 필요 시 블러 반경을 낮춰 보정한다(범위 밖 표면은 그대로).
3. 테두리 — 카드 테두리가 세로 변(좌우)에서 흐리게 보이던 것을 네 변 고르게 선명하도록 테두리 대비를 높인다.
4. 톤 유지 — 팔레트·브랜드 이미지·문구·기존 글래스 토큰은 그대로 재사용한다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 여행 상세 콘텐츠가 유리로 보인다 (Priority: P1)

여행 상세 화면에서 활동 카드와 주간 달력 래퍼가 반투명 유리로 보여, 이 화면에서도 글래스 디자인이 체감된다.

**Why this priority**: 사용자가 가장 자주 보는 화면이 여행 상세다. 여기서 글래스가 보이지 않으면 "전면 적용"이 성립하지 않는다.

**Independent Test**: `ActivityCard`가 글래스 변형으로 렌더되고, 주간 달력 래퍼가 글래스 유틸 클래스를 갖는지로 검증한다.

**Acceptance Scenarios**:

1. **Given** 여행 상세, **When** 활동 카드가 렌더되면, **Then** 반투명 유리 표면으로 보인다.
2. **Given** 여행 상세(모바일), **When** 주간 달력이 렌더되면, **Then** 래퍼가 반투명 유리 표면으로 보인다.
3. **Given** 긴 일정 목록, **When** 스크롤하면, **Then** 눈에 띄는 끊김이 없다(실기기 정본).

---

### User Story 2 - 카드 테두리가 네 변 고르게 선명하다 (Priority: P2)

카드 테두리가 상하좌우 네 변에서 같은 세기로 선명하게 보여, 좌우가 흐려 깨져 보이던 인상이 사라진다.

**Why this priority**: 테두리 비대칭은 카드가 열려 있는 것처럼 보여 완성도를 떨어뜨린다.

**Independent Test**: 카드 테두리 대비 토큰/클래스가 이전보다 높은 값으로 설정됐는지로 검증한다.

**Acceptance Scenarios**:

1. **Given** 임의 카드, **When** 렌더되면, **Then** 네 변 테두리가 같은 세기로 보인다.

---

### Edge Cases

- 저사양 기기 스크롤 — 활동 카드 블러가 겹칠 때 성능 저하 시, 블러 반경을 낮춘다.
- 활동 카드 내부의 카테고리 배지·현금 배지 대비 — 반투명 배경 위에서도 4.5:1 유지.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 활동 카드(`ActivityCard`)는 글래스 표면으로 렌더되어야 한다.
- **FR-002**: 여행 상세 주간 달력 래퍼는 글래스 표면으로 렌더되어야 한다(달력 셀 텍스트 가독성은 유지).
- **FR-003**: 글래스 값은 기존 `:root` 글래스 토큰을 재사용해야 한다(신규 하드코딩 색 금지).
- **FR-004**: 카드 테두리는 네 변에서 같은 세기로 선명해야 한다(테두리 대비 상향).
- **FR-005**: 글래스 표면 위 본문·배지 텍스트는 WCAG AA(4.5:1)를 유지해야 한다.
- **FR-006**: 팔레트·브랜드 이미지·문구·기존 글래스 유틸은 변경하지 않는다.

### Key Entities

해당 없음 — 영속 데이터 스키마 변경 없음.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 여행 상세 화면에서 활동 카드·주간 달력 래퍼가 글래스 표면으로 렌더된다.
- **SC-002**: 카드 테두리 네 변이 같은 세기로 보인다.
- **SC-003**: 글래스 표면 위 텍스트·배지 대비 4.5:1 이상.
- **SC-004**: 신규 하드코딩 색 0건(기존 토큰 재사용).
