# Feature Specification: 로그인 후 홈 캘린더 제거

**Feature Branch**: `039-home-remove-calendar`  
**Created**: 2026-06-02  
**Status**: Draft  
**Input**: User description: "로그인 후 홈(여행 목록)에서 상단 월 캘린더 제거, 여행 목록 카드만 노출"

## Clarifications

1. **홈의 역할 축소** — 로그인 후 홈은 "내 여행 목록"만 보여주면 충분하다. 통합 캘린더(여러 여행의 일자를 한 달력에 모아 보는 뷰)는 홈에서 제거한다.
2. **dead code 정리** — 통합 캘린더가 홈에서만 쓰였다면 컴포넌트도 함께 제거해 잔재를 남기지 않는다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 홈에서 여행 목록만 본다 (Priority: P1)

로그인한 사용자가 홈에 들어가면 상단 월 캘린더 없이 자기 여행 목록 카드만 본다. 화면이 단순해지고 목록 진입이 빨라진다.

**Why this priority**: 통합 캘린더는 정보 밀도만 높이고 홈의 1차 목적(여행 선택)을 흐린다. 제거가 이 feature의 전부다.

**Independent Test**: 홈을 열어 캘린더 영역이 없고 여행 목록 카드가 단일 컬럼으로 나열되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 여행이 1개 이상인 사용자, **When** 홈에 진입하면, **Then** 상단 월 캘린더가 보이지 않고 여행 목록 카드만 나열된다.
2. **Given** 여행이 0개인 사용자, **When** 홈에 진입하면, **Then** 캘린더 없이 "등록된 여행이 없습니다" 안내만 보인다.

### Edge Cases

- 여행이 많을 때 목록은? → 단일 컬럼으로 세로 나열(기존 카드 형식 유지).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 홈(여행 목록 페이지)은 상단 통합 월 캘린더를 렌더하지 않아야 한다.
- **FR-002**: 홈은 사용자의 여행 목록 카드를 단일 컬럼으로 나열해야 한다(기존 카드 내용·기간·역할 표기 유지).
- **FR-003**: 통합 캘린더 전용 컴포넌트·데이터 조회가 홈에서만 쓰였다면 제거하여 미사용 코드를 남기지 않아야 한다.
- **FR-004**: 기존 자동화 테스트는 변경된 레이아웃에 맞게 갱신되어 전부 통과해야 한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 홈 렌더 결과에 월 캘린더 그리드가 존재하지 않는다.
- **SC-002**: 여행 목록 카드는 그대로 노출되고 카드에서 상세로 진입할 수 있다.
- **SC-003**: 전체 테스트가 통과한다(레이아웃 검증 테스트 갱신 포함).
