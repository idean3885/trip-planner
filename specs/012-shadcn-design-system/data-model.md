# Phase 1: Data Model — 디자인 시스템 기반 제정 (012)

**Feature**: 012-shadcn-design-system
**Date**: 2026-04-19

본 피처는 DB 스키마 변경이 없다. 대신 **디자인 토큰**과 **컴포넌트 variants 규약**이 데이터 모델 역할을 한다. 모두 빌드 시점 정적 값이며 런타임 상태를 보유하지 않는다.

## 1. DesignToken (W3C DTCG 호환)

**Location**: `design/tokens.json`
**Shape**: W3C Design Tokens Community Group 스펙 v1 호환 JSON. 편집은 디자이너(Figma Tokens Studio) 또는 개발자 수기.

### 카테고리

| 카테고리 | 필수 | 설명 | 대응 CSS 변수 prefix |
|----------|------|------|---------------------|
| `color` | ✓ | 색상 팔레트(primary/surface/sky 등) | `--color-*` |
| `spacing` | ✓ | 간격 스케일(4px 단위) | `--spacing-*` |
| `radius` | ✓ | 모서리 반경 | `--radius-*` |
| `shadow` | ✓ | 그림자 프리셋(card·card-hover·fab) | `--shadow-*` |
| `fontFamily` | ✓ | 기본·코드 폰트 스택 | `--font-*` |
| `fontSize` | ✓ | 타이포 스케일 | `--text-*` |
| `lineHeight` | ✓ | 라인 높이 스케일 | `--leading-*` |

### 파일 스키마 (단순화 예시)

```json
{
  "$schema": "https://tokens.studio/schemas/design-tokens.json",
  "color": {
    "primary": {
      "50": { "$value": "#eff8ff", "$type": "color" },
      "500": { "$value": "#0e90e0", "$type": "color" },
      "900": { "$value": "#0c4169", "$type": "color" }
    },
    "surface": {
      "0":   { "$value": "#ffffff", "$type": "color" },
      "500": { "$value": "#64748b", "$type": "color" },
      "900": { "$value": "#0f172a", "$type": "color" }
    },
    "background": { "$value": "{color.surface.0}", "$type": "color" },
    "foreground": { "$value": "{color.surface.900}", "$type": "color" }
  },
  "spacing": {
    "content-max": { "$value": "40rem", "$type": "dimension" }
  },
  "radius": {
    "card": { "$value": "0.625rem", "$type": "dimension" }
  },
  "shadow": {
    "card": {
      "$value": "0 0 0 1px rgb(15 23 42 / 0.08), 0 1px 3px rgb(15 23 42 / 0.06)",
      "$type": "shadow"
    }
  }
}
```

### Validation Rules

- 필수 카테고리 누락: Style Dictionary 파싱 실패 → `pnpm run tokens:build` 종료 코드 ≠ 0.
- `$value`/`$type` 페어가 아닌 노드: DTCG 스펙 위반으로 빌드 실패.
- 참조 문법(`{color.surface.0}`) 순환: 빌드 실패.
- 값 타입 불일치(예: color에 dimension 값): 빌드 실패.

### State Transitions

없음. 빌드 입력 → `@theme` CSS 변수 산출물로 1회 변환.

## 2. CSS Theme Variables (`@theme` in `globals.css`)

**Location**: `src/app/globals.css`의 `@theme { … }` 블록.
**Source**: `design/tokens.json`에서 `pnpm run tokens:build`로 자동 생성. 파일의 `/* BEGIN:tokens */`와 `/* END:tokens */` sentinel 사이만 재작성된다.

### 구조 (개념)

```css
@import "tailwindcss";
@plugin "@tailwindcss/typography";

@theme {
  /* BEGIN:tokens (generated by scripts/build-tokens.ts) */
  --color-primary-50: #eff8ff;
  --color-primary-500: #0e90e0;
  --color-surface-0: #ffffff;
  --color-background: var(--color-surface-0);
  --color-foreground: var(--color-surface-900);
  --spacing-content-max: 40rem;
  --radius-card: 0.625rem;
  --shadow-card: 0 0 0 1px rgb(15 23 42 / 0.08), 0 1px 3px rgb(15 23 42 / 0.06);
  /* END:tokens */

  /* 수동 정의(제너레이터가 덮지 않음) */
  --font-sans: ui-sans-serif, system-ui, -apple-system, …;
}

/* 기존 .prose .table-cards, .weather-toggle 등 그대로 유지 */
```

### Validation Rules

- sentinel 블록 밖의 수동 정의는 토큰 빌드로 덮이지 않아야 한다(scripts/build-tokens.ts에서 검증).
- shadcn이 요구하는 최소 변수 키(`--color-background`, `--color-foreground`, `--color-border`, `--color-ring` 등)는 `design/tokens.json`에서 반드시 alias로라도 정의되어야 한다(누락 시 shadcn 컴포넌트가 투명 배경으로 렌더).

## 3. Component Variants 규약 (class-variance-authority)

**Location**: `src/components/ui/*.tsx` 내부.
**Library**: `class-variance-authority@^0.7`

### 정의 규약

- 모든 vendored shadcn 컴포넌트는 `const xxxVariants = cva(base, { variants, defaultVariants })` 패턴을 따른다.
- `variants` 키는 semantic 속성(`variant`, `size` 등)만 사용. 색상 값 직접 매핑 금지(토큰 경유).
- `className` prop은 `cn(xxxVariants({ … }), className)`으로 병합. 사용처에서 override 가능.
- 마이그레이션 대상 폼 6종(`ActivityForm`·`AuthButton`·`DeleteTripButton`·`LeaveTripButton`·`InviteButton`·`TodayButton`)은 내부적으로 `Button`·`Input` 등 vendored 컴포넌트를 조합하되, **자체 variants를 새로 정의하지 않는다**(복잡도 억제).

### 예시 — Button (shadcn 기본)

```tsx
const buttonVariants = cva(
  "inline-flex items-center justify-center rounded-md text-sm font-medium …",
  {
    variants: {
      variant: {
        default: "bg-primary text-primary-foreground hover:bg-primary/90",
        destructive: "bg-destructive text-destructive-foreground …",
        outline: "border border-input bg-background hover:bg-accent …",
        ghost: "hover:bg-accent hover:text-accent-foreground",
        link: "text-primary underline-offset-4 hover:underline",
      },
      size: { default: "h-10 px-4 py-2", sm: "h-9 px-3", lg: "h-11 px-8", icon: "h-10 w-10" },
    },
    defaultVariants: { variant: "default", size: "default" },
  }
);
```

### Validation

- 타입: cva가 반환하는 `VariantProps<typeof xxxVariants>`를 컴포넌트 props 타입에 interpolate. `tsc`가 허용되지 않은 variant 값을 차단.
- 라이트 단독 정책: `dark:` 접두어 클래스는 허용하되 `@theme`의 다크 변수가 부재해 실질 변화 없음. shadcn 템플릿의 `dark:` 문자열을 제거하는 수작업은 선택적(PR3 이후 별도 cleanup 이슈).

## 4. DesignHandoff Issue (GitHub Issue Forms payload)

**Location**: `.github/ISSUE_TEMPLATE/design-handoff.yml`에 스키마 정의, 실 인스턴스는 GitHub Issues.
**Required Fields** (R-7 참조): Figma URL, 스크린샷, Variants & States, 인터랙션, 데이터 바인딩, 토큰 변경 여부.

### 상태 전이

- **Open (handoff 접수)**: 디자이너 생성 시점. `design-handoff` 라벨 자동 부여.
- **Assigned**: 개발자가 본인 assign + 세션 시작.
- **In PR**: 연결된 feature PR이 생성되면 이슈 본문에 `Closes #NNN` 또는 `Handoff: #NNN` 역참조.
- **Closed**: PR 머지 시 자동 close(GitHub 기본 규약).

본 데이터 모델은 GitHub가 소유. 본 피처는 **템플릿 스키마만 정의**하고 인스턴스 생성·전이는 운영 단계의 일.
