# Feature Specification: 활동 상세·편집 UX 정비

## 배경

활동 카드 본문을 탭하면 곧장 편집 폼이 펼쳐져, 내용만 보려 해도 의도치 않게 편집에
진입한다(특히 작은 화면). 폼 레이아웃은 핵심 정보(비용·통화·예약)가 메모 아래에
떨어져 있고 메모 영역이 좁아 잘려 보인다. 상세 안 링크는 새 탭으로 열려 화면 맥락을
벗어난다.

## User Scenarios

### US1 — 카드 탭 → 상세 → 편집 2단계 (P1)

활동 카드를 탭하면 편집과 같은 레이아웃의 **상세 보기(읽기 전용)**가 펼쳐진다.
상세 안의 "편집" 버튼을 눌러야 입력이 활성화된다.

**Acceptance**
- 카드 본문을 탭하면 읽기 전용 상세가 펼쳐진다(입력 비활성).
- 상세의 "편집"을 누르면 같은 자리에서 입력이 활성화된다.
- 상세에서 "닫기"를 누르면 카드 표시로 돌아간다.
- 편집 권한이 없으면 상세는 보되 편집 버튼은 노출하지 않는다.

### US2 — 폼 레이아웃 정비 (P1)

상세·편집 폼에서 핵심 정보를 위쪽에 모으고 메모를 읽을 수 있는 높이로 둔다.

**Acceptance**
- 비용·통화·예약 여부가 메모 위에 온다.
- 상세를 펼쳤을 때 메모 영역이 잘리지 않을 최소 높이를 확보한다.
- 장소 입력은 메모와 구분되도록 라벨·도움말로 역할을 명확히 한다.

### US3 — 상세 내 링크 팝업 (P2)

상세·카드의 링크를 클릭하면 새 탭 대신 팝업 창으로 연다.

**Acceptance**
- 링크 클릭 시 팝업 창으로 열린다(브라우저/앱 맥락 유지).
- 링크 클릭이 카드 탭(상세 펼침)으로 번지지 않는다.

## Requirements

- **FR-001** 카드 본문 탭은 읽기 전용 상세를 펼친다(편집 직행 금지).
- **FR-002** 상세의 "편집" 버튼으로만 입력을 활성화한다.
- **FR-003** 상세는 "닫기"로 카드 표시로 복귀한다.
- **FR-004** 비용·통화·예약을 메모 위로 배치한다.
- **FR-005** 상세 펼침 시 메모 영역에 최소 높이를 확보한다.
- **FR-006** 장소 입력의 역할을 라벨·도움말로 명확히 한다.
- **FR-007** 상세·카드 링크는 팝업 창으로 열고, 클릭이 상위 탭으로 전파되지 않는다.

## 장소/메모 필드 — 통합 여부 결정

선행 검증(코드 기준): 장소(`location`)는 Google 캘린더 동기화의 위치 필드, ICS
`LOCATION`, 구글맵 검색 링크 생성에 쓰인다. 캘린더·지도 연동으로 실제 의미가 있어
**두 필드를 유지**한다. 다만 장소 입력에 라벨·도움말을 붙여 메모와의 차이를 분명히
한다. (실 캘린더/지도에서의 최종 표시는 운영 환경에서 별도 확인.)

## Out of Scope

- 활동 데이터 스키마 변경(장소·메모 필드 통합/추가 없음).
- 캘린더 동기화 동작 변경.

## 용어

- **상세 보기**: 편집과 같은 레이아웃의 읽기 전용 펼침 상태.
- **편집 모드**: 입력이 활성화된 폼.
