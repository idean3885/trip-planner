## QS1. 여행 요소 검색·추천 (US1)

### 시나리오: 마드리드 숙소 검색 (MCP)

```
사용자: "마드리드 6/11~12 숙소 추천해줘"
```

**예상 동작**:
1. Claude가 `search_destinations("Madrid")` MCP 호출 → 목적지 ID 획득
2. Claude가 `get_hotels(destination_id, "2026-06-11", "2026-06-12")` MCP 호출 → 호텔 목록
3. Claude가 결과를 분석하여 추천 기준(위치, 가격, 리뷰)과 함께 3~5개 후보 제시

**검증 포인트**:
- [ ] MCP 도구 호출이 성공하는가
- [ ] 호텔 목록에 이름, 가격, 리뷰 점수가 포함되는가
- [ ] Claude가 추천 기준을 함께 제시하는가
- [ ] 사용자가 후보 중 선택할 수 있는 형태인가

### 시나리오: 바르셀로나 관광지 검색 (CLI)

```bash
python scripts/search_attraction_locations.py --query "Barcelona"
python scripts/search_attractions.py --location-id "{위에서 얻은 ID}" --date "2026-06-18"
```

**예상 동작**:
1. `search_attraction_locations` 실행 → 위치 ID 획득
2. `search_attractions` 실행 → 관광지 목록 stdout 출력
3. Claude가 결과를 분석하여 추천 제시

**검증 포인트**:
- [ ] CLI 스크립트가 정상 실행되는가
- [ ] 관광지 이름, 가격, 소요시간이 포함되는가
- [ ] 예약 필요 여부가 안내되는가

### 시나리오: 식당 추천

```
사용자: "리스본 알파마 지구 근처 저녁 식당 추천해줘"
```

**예상 동작**:
1. Claude가 웹 검색으로 식당 후보 탐색
2. 최신 리뷰를 크로스체크하여 운영 여부 확인
3. 추천 기준(위치, 메뉴, 가격대, 리뷰)과 함께 제시

**검증 포인트**:
- [ ] 웹 검색 결과에 기반한 구체적인 식당이 추천되는가
- [ ] 운영 여부에 대한 언급이 있는가 (최신 리뷰 근거)
- [ ] 예약 필요 여부가 안내되는가

## QS2. 일정 생성 (US2)

### 시나리오: 1일차 일정 생성

```
사용자: "6/7 리스본 도착일 일정 만들어줘. 숙소는 [확정된 숙소명]으로 예약 완료."
```

**예상 동작**:
1. Claude가 기존 예약 데이터(숙소)를 유지
2. 공항→숙소 이동, 저녁 식사, 근처 산책 코스를 Claude가 웹 검색으로 채움
3. `daily/day01-0607-lisbon.md` 생성

**검증 포인트**:
- [ ] CLAUDE.md에 정의된 데일리 파일 포맷(7개 섹션)을 따르는가
- [ ] 기존 예약 완료 숙소가 `✅ 예약 완료`로 유지되는가
- [ ] 미확정 부분이 새로 채워져 있는가
- [ ] 예약 필요 여부가 각 항목에 표기되는가

### 시나리오: 전체 일정 요약 생성

```
사용자: "전체 일정 요약 만들어줘"
```

**예상 동작**:
1. Claude가 모든 daily 파일을 읽고 `itinerary.md` 생성
2. 여행 루트·기간·숙소 한눈에 보기 포함
3. 일별 목차로 각 daily 파일 링크

**검증 포인트**:
- [ ] 여행 루트·기간·숙소 요약이 포함되는가
- [ ] 일별 목차에서 daily 파일로 링크가 연결되는가
- [ ] 핵심 정보(숙소, 이동, 관광, 식사, 비용)가 요약에 포함되는가

## QS3. 웹앱 딜리버리 (US3)

### 시나리오: AppPaaS 배포 후 모바일 열람

```
개발자: 웹앱 빌드 → AppPaaS 배포
동행자: 공유받은 URL을 모바일에서 열기
```

**예상 동작**:
1. Next.js 앱 빌드 → AppPaaS 배포
2. 웹앱 URL 접근 가능
3. 모바일 브라우저에서 정상 표시

**검증 포인트**:
- [ ] AppPaaS 배포가 정상 완료되는가
- [ ] 모바일에서 추가 조작(줌, 가로 스크롤) 없이 열람 가능한가
- [ ] 일별 일정 링크가 모바일에서 탭으로 이동 가능한가
- [ ] 숙소/관광지 외부 링크가 모바일에서 정상 동작하는가

### 시나리오: 일정 수정 후 반영

```
개발자: 일정 수정 → 앱 재배포
동행자: 모바일에서 새로고침
```

**검증 포인트**:
- [ ] 수정된 내용이 웹앱에 반영되는가
- [ ] 동행자가 새로고침으로 최신 일정을 확인할 수 있는가

## Edge Case 검증

| 케이스 | 검증 방법 |
|--------|----------|
| 검색 결과 0건 | `python scripts/search_attraction_locations.py --query "xyzabc123"` → 안내 메시지 확인 |
| 기존 예약 vs 검색 충돌 | 예약 완료 숙소가 있는 상태에서 같은 날짜 숙소 검색 → 기존 예약 유지 확인 |
| API 타임아웃 | 네트워크 차단 상태에서 CLI 스크립트 실행 → 에러 메시지 확인 |
