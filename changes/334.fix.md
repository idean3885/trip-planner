**구글 동의 화면 취소 시 "Server error" 대신 친절한 안내 (#334)**: Google OAuth 동의 화면에서 사용자가 "취소"하면 Google이 `iss` 없는 error 응답을 보내고 Auth.js v5가 이를 Configuration 에러로 승격, 기본 `/api/auth/error` 페이지의 "Server error" 문구로 이어지는 UX 회귀가 있었다. `auth.config.ts`에 `pages.error = "/auth/signin"`을 지정해 에러 시점에도 signin 페이지로 라우팅하고, signin 페이지가 `?error=<code>`를 해석해 "로그인을 취소했습니다. 다시 시도해 주세요" 같은 맥락 안내를 노출. 사용자가 취소한 동작이 서버 에러 화면으로 보이지 않는다.
