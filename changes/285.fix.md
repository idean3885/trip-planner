**DAY 0 노출 해소 — sortOrder 데이터 보정 + API 자동 채번**: 기존 DB의 `Day.sort_order = 0` 레코드들을 1회성 마이그레이션(`20260419140000_backfill_day_sortorder_285`)으로 각 Trip별 `date ASC` 순 1~N 재번호. POST/PUT/DELETE `/api/trips/:id/days`는 이제 `sortOrder`를 서버가 자동 관리하며 클라이언트 값은 무시. `src/lib/day-order.ts`의 `resortDaysByDate(tx, tripId)`가 transaction 내에서 전체 재정렬을 수행하여 중간 삽입·날짜 변경·삭제 후에도 DAY 1부터 순서 보장. Day.sortOrder 컬럼 폐기 등 구조적 재설계는 별도 [#296](https://github.com/idean3885/trip-planner/issues/296) (v2.5)에서 무중단으로 진행.
