**활동 생성/수정/삭제 실패 시 에러 토스트 표시**: 이전엔 API 실패가 async throw로 unhandled rejection 발생(Vitest 4 + GitHub Actions reporter에서 `##[error]` 집계로 CI test step을 fail시키는 2차 피해). 이제 `ActivityList`의 3종 handler가 `try/catch`로 감싸 `sonner` 토스트(`toast.error("활동 … 에 실패했습니다")`)로 사용자에게 알리고 조용히 복귀. 테스트도 throw 기대 → toast 호출 관찰로 재설계. `.github/workflows/ci.yml`의 test step은 `continue-on-error`를 제거하여 blocking 게이트로 전환.
