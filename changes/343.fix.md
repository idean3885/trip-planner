**GCal scope를 `calendar.events` → `calendar` 전체로 확장해 DEDICATED 모드 복구 (#343)**: `calendars.insert`(전용 캘린더 자동 생성)는 `calendar` 또는 `calendar.app.created` scope를 요구해 기존 `calendar.events` 단독으론 403으로 실패하던 UX를 해소. Testing 모드에서는 Test users 한정 운영이라 scope 확대가 외부 악용 경로를 넓히지 않는다는 판단(유료 Production 승급 보류 유지). 기존에 `calendar.events`로 동의해 둔 사용자는 `hasCalendarScope`가 legacy 값을 여전히 유효 권한으로 인정해 즉시 재동의를 강요받지 않는다. 다만 DEDICATED를 쓰려면 재동의가 한 번 필요.
